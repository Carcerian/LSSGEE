================================================================================
v1.1 "Enhanced EE" (Early 2025, ~Q1)
================================================================================
Timeline: 6-8 weeks after v1.0 release
Focus: Complete the SQLite feature set for persistent data

NEW FEATURES
-----------

1. Object-Scoped SQLite Queries
   - Persistent data tied to specific creatures/placeables
   - Survive creature death (stored externally, restored on respawn)
   - Use case: NPC relationship tracking, quest progress per target
   - UI: Radio button option (Object / Module / Campaign scope)
   - Complexity: Medium (requires object reference tracking)

2. UUID Lookup Generator
   - Generate unique IDs for persistent object identification
   - Assign UUID to creature on spawn
   - Store UUID → object mapping in database
   - Retrieve object by UUID across saves/transitions
   - Use case: Quest markers, companion tracking, persistent relationships
   - Output: Clean generator with one click → add UUID assignment to scripts
   - Complexity: Medium

3. Effects Fold-In
   - VFX constant additions to existing effect generators
   - Add visual effect type dropdowns to "Apply Effect" generator
   - Pre-built combinations (e.g., "Healing + Blue Light", "Damage + Red Spark")
   - Use case: Faster effect creation with visual feedback
   - Complexity: Low (data-driven, UI extension)

4. Enhanced Documentation
   - Video tutorial: Using SQLiteGen (5-10 min)
   - Database schema reference: How persistent storage works
   - Advanced guide: Combining SQLiteGen + Carcerian's Tailor
   - FAQ: Common questions about scope and persistence
   - Complexity: Low (documentation only)

5. Linux Binary Distribution
   - Cross-compile scriptgenerator.exe for Linux
   - Distribute as AppImage or native binary
   - No feature changes, binary-only
   - Complexity: Medium (build infrastructure)

TECHNICAL WORK
--------------
- Refactor SQLite generator into reusable components
- Extend constants.pas with UUID constant family (if needed)
- Add object-reference tracking unit
- Linux build pipeline setup (Lazarus → AppImage)

TESTING
-------
- Module-scope SQLite: existing tests remain valid
- Campaign-scope SQLite: existing tests remain valid
- Object-scope SQLite: new test coverage (create/query/cleanup)
- UUID generator: create/retrieve/persist cycle validation
- Effects fold-in: verify generated code compiles and runs

RELEASE CRITERIA
----------------
- All tests pass (Windows, Linux)
- Documentation complete and reviewed
- One community tester confirms real-world usage
- No regressions in v1.0 generators

================================================================================
v2.0 "The Platform" (Mid 2025, ~Q2-Q3)
================================================================================
Timeline: 8-12 weeks after v1.1 release
Focus: Major architectural expansion (NUI, JSON, schema builder)

NEW FEATURES
-----------

1. NUI Popup Generator ⭐ (Major Feature)
   - Wait for: NUI API v1.0 stability (minimum 6 months post-v1.1)
   - Generate: PopupWindow, PopupMenu, PopupChart, PopupForm scripts
   - UI: Visual builder for popup layouts (drag-drop controls)
   - Output: Complete NWScript NUI initialization and binding code
   - Use case: Advanced UIs without hand-coding NUI bindings
   - Complexity: High (entirely new subsystem)

2. JSON Serialization Generator
   - Generate: JSON-based data export/import
   - Use case: Export player data to JSON, reload custom configs
   - Functions: GenerateJSON(), ParseJSON() helpers
   - Complexity: Medium (JSON library integration)

3. SQLite Schema Builder ⭐ (Major Feature)
   - GUI tool: Define tables, columns, constraints visually
   - Output: DDL script to initialize databases
   - Use case: Design custom persistent storage schemas without SQL
   - Complexity: High (UI for database design)

4. Visual Theme Customization
   - Color picker: Change UI colors (lilac → custom)
   - Font selector: Choose code font, size, style
   - Theme save/load: User preferences persistent
   - Built-in themes: Dark mode, light mode, classic
   - Complexity: Low (UI framework feature)

5. macOS Native Binary
   - Compile for macOS via Lazarus
   - Code-sign and notarize
   - Distribute via DMG installer
   - Complexity: Medium (macOS build infrastructure)

ARCHITECTURAL CHANGES
---------------------
- Plugin system: Allow third-party generator packages
- Extended generator API: Base classes for custom generators
- Settings persistence: User preferences and recent projects
- Project file format: Save generator configurations for reuse

TESTING
-------
- NUI generator: Full NUI binding cycle with test popups
- JSON generator: Serialization round-trip validation
- Schema builder: DDL generation and database initialization
- Theme system: All themes render correctly across platforms
- macOS binary: Native compilation and runtime testing

RELEASE CRITERIA
----------------
- NUI API confirmed stable (minimum 6 months real-world usage)
- All tests pass (Windows, Linux, macOS)
- Plugin system documented with example
- One community tester creates custom plugin successfully

================================================================================
PHILOSOPHY
================================================================================

Arcana follows Lilac Soul's original vision:
  1. **Accessibility:** Novice builders should generate correct NWScript
  2. **Transparency:** Generated code should be readable and understandable
  3. **Extensibility:** Advanced users can modify and extend generators
  4. **Stability:** Core features work reliably; new features tested thoroughly
  5. **Community:** Tool is free, open source, belongs to the community

This roadmap serves the community, not vice versa. Priorities adjust based on
what builders actually need and use.
