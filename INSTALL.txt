ARCANA v1.0 — INSTALLATION & SETUP GUIDE
==========================================

Quick Start (Users)
===================

STEP 1: Locate the executable
  - Unzip the distribution
  - Navigate to: linux/ folder
  - Find: scriptgenerator.exe
  
  Note: The folder is named "linux" for legacy reasons (The Krit's original
  build path). This is the Windows executable and runs fine on Windows.

STEP 2: Run for the first time
  - Double-click scriptgenerator.exe
  - On first launch, you'll be prompted for your NWN:EE installation folder
  - Navigate to your main NWN directory (where nwnmain.exe is located)
  - Click OK
  
  Typical paths:
    C:\Program Files\Neverwinter Nights Enhanced Edition\
    C:\Program Files (x86)\Neverwinter Nights Enhanced Edition\
    C:\GOG Games\Neverwinter Nights Enhanced Edition\

STEP 3: Choose a generator
  - Main menu shows 30+ generators
  - Click any generator name to open its form
  - Configure your script (dropdowns, text fields, buttons)
  - Click "Add to Script" or "Next" to generate NWScript code
  - Code appears in the main window's text area

STEP 4: Save and test
  - File menu → "Save Script"
  - Choose a .nss filename
  - Place in your module's scripts folder
  - Test compile in your NWN module editor
  - If code errors: report with exact error message

TROUBLESHOOTING
---------------

Q: "NWN installation not found"
A: The tool asks for your main NWN:EE folder on first run.
   Options → "Locate Neverwinter Nights manually" to reconfigure.
   Point to the folder containing nwnmain.exe.

Q: "Script won't compile in NWN"
A: Generated code is syntactically correct. Check that:
   1. You're using NWN:EE (original NWN1 may be missing functions)
   2. Function exists in your NWN version (not all feats/spells in basic install)
   3. Syntax matches NWScript conventions (copy into script editor, not .nss file directly)

Q: "I don't recognize a function in the generated code"
A: Check NWScript documentation or CHANGELOG.txt in this distribution.
   Most functions are from Bioware's NWN API. SQLiteGen functions (SqlPrepareQuery, etc.)
   require EE version 8.320.39 or later.

Q: "Can I modify the generated code?"
A: Yes! Generated code is just NWScript. Edit it freely to suit your needs.
   The generator creates a good starting point; customize as needed.

For Developers
==============

BUILD FROM SOURCE
-----------------

Requirements:
  - Lazarus IDE 4.8 or later (https://www.lazarus-ide.org)
  - Free Pascal (FPC) 3.2.2 (included with Lazarus)
  - Windows, Linux, or macOS

Steps:
  1. Extract the zip to a folder
  2. Open Scriptgen.lpi in Lazarus
  3. Project → Build Project (or Ctrl+F9)
  4. Output: linux/scriptgenerator.exe
  
  Note: Output folder name ("linux") is legacy; use any OS.

Cross-Platform Compilation:
  - Windows to Linux: Requires Linux cross-compiler (complex setup)
  - Windows to macOS: Requires macOS SDK (Apple developer tools)
  - For v1.0, recommend compiling on target OS when possible

Custom Generators:
  - Each generator is a .pas unit with a corresponding .lfm form
  - Pattern: [name].pas (logic) + [name].lfm (UI)
  - Register in Scriptgen.lpr uses clause
  - See sqlitegen.pas and sqlitegen.lfm for v1.0 example
  - Documentation: README.md section on other generators

GETTING STARTED
===============

If this is your first time:
  1. Read README.md (5-10 minutes)
  2. Launch scriptgenerator.exe
  3. Click "SQLite Persistent Value" (newest feature, good intro)
  4. Fill in: Scope → Key → Value
  5. Click "Set Value" to generate code
  6. Review generated code in the preview
  7. Click "Add to Script" to use it

Then explore:
  - Other generators in the menu
  - Try "Apply Effect" (classic generator)
  - Try "Spawn Creature" (versatile)
  - Experiment and have fun

Questions?
  - Each generator has built-in help
  - README.md has detailed walkthroughs
  - ROADMAP.txt explains what's coming next
